Estrutura dos grupos

Cada grupo terá 6 integrantes, e cada um assume um papel principal (mas todos podem colaborar em várias áreas). A ideia é refletir a realidade de uma equipe multidisciplinar em um projeto IIoT + Mobile + Cloud.

Importante: os papéis não são "silos". Todos devem se comunicar, colaborar e entender o projeto como um todo. Os papéis apenas definem responsabilidades primárias.
🎯 Papel 1 – Líder de Projeto & Cultura Organizacional

Responsável principal por:

  • Organizar o trabalho do grupo, reuniões, prazos e comunicação interna.
  • Garantir que as decisões técnicas estejam alinhadas ao problema da empresa XYZ (monitorar temperatura, botão, LED, inclusão social).
  • Liderar a parte de Gestão de Pessoas e Cultura Organizacional:
    • Como a solução ajuda operadores e supervisores?
    • Como incentiva o registro de ocorrências (uso do botão, app)?
    • Como é pensada para inclusão (linguagem, acessibilidade)?
  • Coordenar a preparação da apresentação final e do vídeo, garantindo que todos participem.

Disciplinas mais ligadas:

  • Gestão de Pessoas e Cultura Organizacional
  • Projeto Integrador III
  • Extensão Universitária III
🏭 Papel 2 – Engenheiro de IIoT / CLP & Interfaces Industriais

Responsável principal por:

  • Configurar o CLP (ou simulador) para:
    • ler a temperatura (entrada analógica);
    • ler o botão (entrada digital);
    • acionar o LED (saída digital).
  • Definir o mapeamento de variáveis e sua publicação via MQTT ou API para a nuvem.
  • Garantir que os sinais no chão de fábrica cheguem corretamente até o broker (Mosquitto ou AWS IoT Core).
  • Apoiar o design do painel/interface industrial (o que faz sentido para o operador enxergar).

Disciplinas mais ligadas:

  • Internet das Coisas (IoT)
  • Interfaces Industriais
☁️ Papel 3 – Especialista em Infraestrutura Cloud & DevOps – Cenário 1 (EC2)

Responsável principal por:

  • Criar e configurar a instância EC2 (Linux) na AWS.
  • Instalar e configurar:
    • Mosquitto (broker MQTT);
    • Node-RED;
    • MySQL.
  • Cuidar de:
    • Segurança básica (portas, usuários, senhas);
    • Scripts de inicialização/instalação quando possível.
  • Liderar as práticas de DevOps no Cenário 1:
    • Versionar arquivos de configuração;
    • Documentar o passo a passo;
    • Apoiar na criação de pipelines simples (se previsto na disciplina DevOps).

Disciplinas mais ligadas:

  • Eletiva II – DevOps
  • Tecnologias de Cloud
  • Internet das Coisas (IoT)
🌐 Papel 4 – Especialista em Cloud PaaS & Integrações – Cenário 2 (AWS IoT)

Responsável principal por:

  • Configurar o AWS IoT Core:
    • Things, certificados, políticas, tópicos.
  • Criar regras IoT (IoT Rules) e/ou funções Lambda para gravar:
    • Temperatura;
    • Eventos de botão;
    • Comandos para LED
    em um banco de dados gerenciado (RDS ou DynamoDB).
  • Configurar o MQTT Dash para:
    • Exibir temperatura;
    • Indicar eventos de botão;
    • Enviar comando para o LED.
  • Documentar as diferenças (técnicas, de esforço) entre o Cenário 1 e o Cenário 2.

Disciplinas mais ligadas:

  • Internet das Coisas (IoT)
  • DevOps (Infrastructure as Code, se quiser)
  • Cloud
  • Aplicações de IA (integrações)
📱 Papel 5 – Desenvolvedor Mobile & UX / Inclusão Social

Responsável principal por:

  • Pensar e/ou implementar o acesso mobile no Cenário 1:
    • Pode ser:
      • Um app mobile (Android/iOS) consumindo API/MQTT;
      • Ou, no mínimo, garantir que o dashboard do Node-RED seja responsivo e usável no celular.
  • Projetar a interface com foco em:
    • Simplicidade (poucos botões, textos claros);
    • Cores e ícones significativos (alarme, normal, atenção);
    • Inclusão social (tamanho de fonte, contraste, símbolos fáceis de entender).
  • Trabalhar junto ao Eng. IIoT para decidir:
    • Que dados aparecem na tela;
    • Como o botão virtual de comando (LED) será apresentado.

Disciplinas mais ligadas:

  • Desenvolvimento Mobile
  • Interfaces Industriais
  • Gestão de Pessoas (inclusão)
🧠 Papel 6 – Especialista em Dados, IA e Estruturas de Dados

Responsável principal por:

  • Definir o modelo lógico de armazenamento de dados (tabelas no MySQL / estrutura no DynamoDB).
  • Decidir e implementar estruturas de dados na aplicação (listas, filas, mapas, históricos) para:
    • Organizar as leituras de temperatura;
    • Guardar eventos de botão;
    • Recuperar dados de forma eficiente para o app/dashboard.
  • Implementar ao menos um recurso simples de IA/análise inteligente, por exemplo:
    • Classificar estado (normal, atenção, crítico) com base em limites;
    • Identificar anomalias (temperatura fora do padrão em relação à média);
    • Fazer uma pequena previsão ou alerta de tendência.
  • Documentar essa lógica e como ela ajuda a tomada de decisão na XYZ.

Disciplinas mais ligadas:

  • Aplicações de Inteligência Artificial
  • Estruturas de Dados
  • Banco de Dados
