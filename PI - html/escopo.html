Escopo técnico
Três variáveis essenciais do piloto
Introdução

A empresa XYZ definiu três variáveis essenciais que devem fazer parte do piloto IIoT. Todos os grupos devem implementar essas três variáveis em ambos os cenários (EC2 e PaaS AWS).

📊 1. Temperatura (entrada analógica)
  • Medida por um sensor conectado ao CLP (sensor de temperatura real ou simulado).
  • Deve ser lida periodicamente (por exemplo, a cada 5 segundos).
  • Enviada para a nuvem (AWS) e registrada em um banco de dados para histórico.
  • Exibida em tempo (quase) real em um dashboard acessível via celular.
  • Deve permitir visualização de:
    • Temperatura atual;
    • Histórico em gráfico ou tabela;
    • Tendências (subindo, estável, descendo).
Exemplo: Sensor lê 65°C → publica via MQTT → chega à nuvem → grava no BD com timestamp → dashboard mostra em tempo real.
🔘 2. Botão de acionamento (entrada digital)
  • Botão físico conectado ao CLP (ou simulado em software).
  • Quando o operador identifica qualquer condição relevante (ruído, cheiro estranho, peça não conforme, necessidade de ajuste), ele pressiona o botão.
  • Cada acionamento deve gerar um evento que:
    • é enviado para a nuvem;
    • é gravado no banco de dados com timestamp;
    • é associado à temperatura e ao estado da máquina naquele momento.
  • Serve como base para:
    • Análise de comportamento da máquina;
    • Incentivar o registro simples de ocorrências;
    • Criar histórico confiável de eventos.
Exemplo: Operador pressiona botão às 14:32:15 → evento é enviado → BD registra: "Evento em 14:32:15, Temp=72°C, Status=Usinando" → dashboard mostra "Ocorrência registrada".
💡 3. LED de sinalização (saída digital)
  • Um LED físico conectado ao CLP (ou simulado em software).
  • Pode ser ligado ou desligado remotamente por comando vindo da nuvem.
  • O aplicativo mobile ou dashboard deve ter um botão virtual que:
    • envia uma mensagem à nuvem (ex.: "Acionar LED de alerta");
    • a nuvem recebe e envia comando ao CLP;
    • o CLP aciona o LED (liga ou desliga).
  • Uso prático:
    • Supervisor confirma uma ação (ex.: "manutenção em andamento", "parada autorizada");
    • Toca no botão no app;
    • LED no painel acende, sinalizando para o operador que a orientação foi recebida;
    • Feedback visual imediato, sem depender de leitura de mensagens de texto.
Exemplo: Supervisor toca em "Acionar LED Manutenção" no app → mensagem vai para nuvem → nuvem envia comando ao CLP → LED acende na máquina → operador vê imediatamente que manutenção foi autorizada.
